简约是怎么一回事？

每当提到简约这个话题的时候，我第一时间想起的就是中学物理课本的第一页的背面的一个小故事，说爱因斯坦晚年最遗憾的就是依旧没有探索出「大统一理论」，即希望能借由单独一种物理理论来合理解释电磁相互作用、强相互作用和弱相互作用导致的物理现象。

这简直，简直太简约了吧（简约通常还会被人误解有太简单的意思，但在这里的意思是说，这简直酷毙了，但酷毙了它明显是三个字，不简约而且还不好听）

简约不是简单，举一个例子就是，假设做一道数学题，呕心沥血费劲心思终于做出来了，却发现结果等于 1。

如果再要详细的铺述开简约的含义，我想到的就是 Apple，苹果它简直就是把简约这个词融汇贯通于自身的最好的例子，从人文、科技、设计、商业、文化等等太多个方面，它把这个词的一撇一捺都写得棒极了。

当然，我可不是果脑残粉，果吹亦或者是果黑，我接触过的或者了解到的关于苹果的一些特别好的用户体验、工业设计、人文关怀方面等等，当然也深恶痛绝地讨厌苹果做的不好的一些方面。但无论如何，它值得的被赞扬的简约这一方面，让人爱不释手，喔，当然也爱不释口。

开头题图的 Apple Store 门

这扇门是苹果在旧金山的一家官方专卖店的门，事实上它可能看起来更像是一堵墙，一堵玻璃墙。没错，它本质上就是一堵玻璃墙，其实它还有两扇较小的门口在店的两侧。而这偌大的一扇门其实又充当的墙的作用，当天气好的时候，店员们就会把这扇大的用来当作墙的门打开，然后就会发现，整个店变成了一个类似于开放式的广场。有种感觉就是你敞开了胸怀，拥抱一切，让人们不由自主地想要走近。下面是这堵墙打开的时候，它成了一扇门。

外表的工业设计

我们每一个人都……不能说是十分注重自己的外表，但肯定的是我们的外表是我们每天都特别注意去维持或者修饰的一个点。我们的穿衣打扮，还有发型设计，在有些时候我们都觉得太重要了。

比如我们参加一个晚会，或者要去见一些重要的人，我们总会稍微修饰一下自己，有个词叫“仪式感”，可以用在这里形容一下。那些产品的工业设计就如同我们的外表的穿衣打扮。我们肯定都希望它是好看的，且合理的。

Apple 其实非常注重工业设计这一方面，尤其是乔布斯在的时候，当然我们看一下苹果的产品线就能够很清楚，Apple 之所以如此注重工业设计，它是离不开乔布斯的，乔布斯在整个 Apple 的企业文化中注入了灵魂泉源般的东西。哪怕是在上世纪工业制造特别弱的时候，电路板只能人工去焊接，乔布斯十分注重的就是在焊接的时候走线不能歪了。

外观设计造型的简约

苹果的整个产品线流程都在向着简约的方向发展，从最令人显眼的外观开始它尽可能的去统一美化一些不重要的点。比如说，电脑都要有显示屏，要有主机还要有键盘、鼠标，那这么多东西实在是太杂乱了，我们能不能把他们统一一下，或者合在一起也行。最初的 iMac 设计就是这样想的

初代 iMac

然后，直到现在的 iMac 造型

iMac 2017

在1997年乔布斯回归 Apple，他砍掉了许许多多的产品，而专注于研发几款主要的产品，把分散的注意力聚焦在一起，来完成一些伟大的产品，1998 年，在砍掉了大量其他产品线后，Apple 大力推出了 iMac G3。在市场上进行差异化竞争，「Think different」这一口号开始变得广为人知。最终这款半透明糖果色外壳的个人电脑在两年内狂销 200 万台，成功令苹果公司扭亏为盈。而彼时乔布斯也转为正式 CEO，重新开始领导苹果，进入了新的职业生涯。

苹果最成功的产品「iPhone」，从一开始就注重了统一性，当所有的手机几乎都能够开后盖换电池的时候，「iPhone」一推出便不能随意开后盖更换电池，虽然因此得到了众多的嘲讽，但时间证明了简约的统一性是正确的。通过统一性来换取更好的用户体验，事实证明了，只要产品做得符合大众的使用习惯，越简单，越方便，去复杂，它就是能够被广泛传播和接受的。

「iPhone」设计之初只有一个主要的「home」键，扔给用户的思想就是：你要返回，对么？按一下这个键就「ok」了，就是这么简单，明白了么。再到现在，连仅剩的一个「home」键也被抛去了，从设计的角度上，简约统一性的角度上，苹果做的没有错。

从我个人的使用角度体验上，虽然按一个东西可能会更有感觉，是从手指尖到脑神经的一种传递的感觉，一开始觉得按「home」键返回远比「滑动返回」爽太多了吧，「滑动」能有什么特别的感觉呢？但当你一旦试过了，你就不想再用回去了，「滑动返回」没有什么太大的触感来回应你的脑，但是，你的眼睛会盯着屏幕的，给你的另一种感觉就是「嗖」的一下子，「喔？这样就返回主菜单了，太快了，我再试一次」，当我第一次玩这个「滑动返回」时候，我玩了十多次，「滑滑滑……」，感觉好简单，有点好玩的意思。显然，你是不可能去按「home」键按十多次还感觉它挺好玩的，那绝对会把手指头按得难受的。

内容层面的简约

我看过的到目前为止的电子产品的公司的官网，他们设计的简直都是一塌糊涂，像「微软」，我在他们官网找一个产品的时候简直能让人头疼死。包括国内的很多手机公司，他们绝对都是把「官网」当做「电商网站」来设计的。

GUI 的变化

苹果的内容层面，从系统的「GUI」设计，GUI 全称是 Graphical User Interface（图形用户界面），它是用户体验设计的组成部分，它的演变诠释着「简约」和「统一化」这一主题。

苹果电脑 GUI 的演变

iOS6 与 iOS7界面

OS X 10.8 与 OS X 10.10界面

首先，统一性肯定是显然的，苹果肯定会统一自家的电脑和手机以及其他产品的设计语言，上面的两幅图片显示了苹果从「拟物化」到「扁平化」的「GUI」设计风格的转变。苹果为什么要这样做，从简约的角度来解释这一问题，恐怕是最好的回答。

有很多人讨论「到底是『拟物化』更好看还是『扁平化』更好看」，从好看的角度，每个人都有自己的审美观念，我们不能强人所难更不能有歧视化，非得说「扁平化」比「拟物化」更好看云云开了。反而从「简约化」的角度讨论「为什么苹果会改变自己的 GUI 设计语言的风格」，「扁平化」去除了「拟物化」很多臃肿繁琐的表现形式，「拟物化」中用的很多的一些「高光」、「倒角」、「渐变」、「映射」、「阴影」等等接近和符合现实事物的设计风格在「扁平化」中得到的大幅度的削减，这样使得的一个好处就是：一眼看去，「扁平化」更加的清新夺目，给人一股子放松欢快感，没有了「拟物化」那种实物和拥挤感，「扁平化」与「拟物化」在同样展现某个界面的时候，「扁平化」的展现会给人造成一种仿佛能看到更多内容的错觉。

拟物化 iTunes

扁平化 Windows store

当然，现在「扁平化」是所有用户交互界面的主流设计，自从苹果在「iOS7」发布之后，「扁平化」这种简单风的设计语言被人们广泛采用，是对整个设计行业的较大规模的推动发展。

但我觉得这并不就意味着「拟物化」不好，我们离不开它，虽然人们一股脑儿的去喜欢「扁平化」的风格，但苹果从没有抛弃「拟物化」，他们在所有适合使用「拟物化」风格的内容上进行了「拟物」风格渲染，其中最最特别明显的一个点就是「渐变色」的使用。

可以看到，界面的边缘这种窗口阴影渐变的使用。包括在「iOS」中，类似的「渐变」、「模糊」「景深关系」也采用了许多细节的方面。

用户体验方面开箱即用的简约

如果要说到软件层面，感受最深的就是制作「幻灯片」的两个软件，一个是苹果开发的「keynote」，另一个是微软开发的「powerpoint」，从使用角度上「keynote」比「powerpoint」容易上手太多了，我记得高中的时候老师教「powerpoint」这个软件，让大家制作「PPT」，我看到软件界面：

给我第一个感觉就是，哇，那么多按钮啊，这玩意可怎么用啊。老师叨叨叨的讲了一节课，然后又演示了一会，然后逐渐就试着去用。

反观「keynote」，它的按钮明显要少很多，它是把所有的用户使用最频繁的功能放在用户最显眼的地方。抛开其他的功能，在第一眼上，「keynote」要赢得不少瞩目。

当再深入使用之后，我就再也不想用「powerpoint」来制作幻灯片，可能不太好的形容就是，用「keynote」来制作幻灯片本身就是另一种享受，你会喜欢上制作的过程。它的动画设计太棒了，还有一些图表等，也做得非常好看。

而且在审美上这也是「微软」的薄弱之处，乔布斯说了很多堵人的话，但他说微软没有品位“微软唯一的问题就是没有品味，完全没有。”这句话，是没错的。

苹果把它的整个「文化」、「思想」、「科技」、「人文」都尽可能的融进它的产品，达到一种让人细微察觉才发现的极简状态，把产品保持一种温度，这是它做的最好的地方。

表达的简约

苹果最擅长的就是从来不用刻意的表达来进行宣传，但它每次宣传出效果都恰如其分的好。比如它要宣传它的相机，它从来不会与人争锋般的堆砌一大堆硬件参数，或者说跟用户解释哪里好，它会拍一些照片，或者拍一段视频来放给你看，而且动画做的特别好。
